# Vexor (MicroService)

Система автоматизации обучения и аттестации персонала для индустрии HoReCa (рестораны, кафе, отели).

## Назначение проекта

Vexor представляет собой Telegram Mini App (TMA) решение, предназначенное для быстрой проверки знаний сотрудников, проведения периодических аттестаций и мониторинга профессионального уровня команды в реальном времени.

## Функциональные возможности

- Обучающие блоки: деление учебного материала на тематические модули с проверочными тестами.
- Мониторинг KPI: отслеживание прогресса обучения каждого сотрудника.
- Рейтинговая система: формирование внутренних списков лидеров для мотивации персонала.
- Генерация документации: автоматическое создание PDF-отчетов с результатами аттестации.
- Интеграция с Telegram: работа в привычном для сотрудников интерфейсе без установки стороннего ПО.

## Технологический стек

- Платформа: Next.js 15 (App Router)
- Стилизация: Tailwind CSS 4, Framer Motion
- Управление состоянием: Jotai, Zustand
- База данных: PostgreSQL (через Drizzle ORM)
- Специфическое ПО: Puppeteer (серверная генерация отчетов)
- Используемые технологии: Node.js, TypeScript, React, Next.js, Tailwind CSS, Framer Motion, Jotai, Zustand, Drizzle ORM, PostgreSQL, Puppeteer, Chart.js, Lucide/HeroIcons, google/genai

## Инструкция по развертыванию

### 0. Подготовка

Необходимо настроить .env файл в корне проекта.

```
DATABASE_URL=
TELEGRAM_BOT_TOKEN=
TELEGRAM_BOT_ADMINISTRATOR_ID=
GOOGLE_GENERATIVE_AI_API_KEY=
# Source database
STARSHIP_ENDPOINT_KEY=
STARSHIP_ENDPOINT_SECRET=
STARSHIP_ENDPOINT_URL=
```

### 1. Установка зависимостей

```bash
pnpm install
```

### 2. Настройка базы данных

Необходимо настроить подключение к PostgreSQL в файле `.env` и применить схему:

```bash
pnpm db:push
```

### 3. Настройка окружения для TMA

Для корректной работы Telegram Mini App требуется HTTPS-соединение. Рекомендуется использовать туннелирование:

```bash
cloudflared tunnel --url http://localhost:3000
```

### 4. Запуск в режиме разработки

```bash
pnpm dev
```
